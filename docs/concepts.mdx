This page explains the core concepts behind **versionguard**.

## Entry points

VersionGuard exposes a few distinct entrypoint modules depending on the kind of work you want to do.

- `src/index.ts` is the main public API surface for validation, sync, doctor, and shared exports.
- `src/feedback/index.ts` contains suggestion builders that turn validation failures into actionable guidance.
- `src/fix/index.ts` contains deterministic auto-remediation helpers for versions, changelog entries, and synced files.
- `src/tag/index.ts` contains release-tag helpers for reading, validating, and creating tags safely.

When you consume the package, these map to the main `versionguard` entrypoint and the focused feedback, fix, and tag entrypoint modules generated from the source tree.
