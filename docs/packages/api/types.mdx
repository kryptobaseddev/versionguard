## ManifestConfig

Configures the version source manifest.

| Property | Type                  | Required | Description                                                                                                                                                                         |
| -------- | --------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `source` | `ManifestSourceType`  | Yes      | Manifest file to read the version from.  Use `'auto'` for file-existence detection or a specific filename.                                                                          |
| `path`   | `string \| undefined` | No       | Dotted key path to the version field within the manifest.  For example `'version'` for package.json, `'package.version'` for Cargo.toml, or `'project.version'` for pyproject.toml. |
| `regex`  | `string \| undefined` | No       | Regex pattern to extract the version from source-code manifests.  Capture group 1 must contain the version string.                                                                  |

## SchemeRules

Configures scheme-level validation rules applied regardless of versioning type.

| Property             | Type       | Required | Description                                                                                                                                                                                    |
| -------------------- | ---------- | -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `maxNumericSegments` | `number`   | Yes      | Maximum number of numeric segments before a warning is emitted.  Convention is 3 (e.g., `YYYY.MM.MICRO`). Formats with 4+ segments (e.g., `YYYY.0M.0D.MICRO`) are valid but trigger a warning. |
| `allowedModifiers`   | `string[]` | Yes      | Allowed pre-release modifier tags.  When set, version modifiers (e.g., `-alpha`, `-rc1`) are validated against this whitelist. An empty array disallows all modifiers.                         |

## CalVerConfig

Configures CalVer validation rules.

| Property                | Type                   | Required | Description                                                              |
| ----------------------- | ---------------------- | -------- | ------------------------------------------------------------------------ |
| `format`                | `CalVerFormat`         | Yes      | Calendar format used when parsing and validating versions.               |
| `preventFutureDates`    | `boolean`              | Yes      | Rejects versions that point to a future date.                            |
| `strictMutualExclusion` | `boolean \| undefined` | No       | Enforces that week tokens (WW/0W) cannot be mixed with month/day tokens. |

## ParsedCalVerFormat

Parsed token layout for a supported CalVer format string.

| Property  | Type                               | Required | Description                                                                                                           |
| --------- | ---------------------------------- | -------- | --------------------------------------------------------------------------------------------------------------------- |
| `year`    | `"YYYY" \| "YY" \| "0Y"`           | Yes      | Year token captured from the format string.                                                                           |
| `month`   | `"MM" \| "M" \| "0M" \| undefined` | No       | Month token captured from the format string when present.                                                             |
| `week`    | `"WW" \| "0W" \| undefined`        | No       | Week token captured from the format string when present.                                                              |
| `day`     | `"DD" \| "D" \| "0D" \| undefined` | No       | Day token captured from the format string when present.                                                               |
| `counter` | `"MICRO" \| "PATCH" \| undefined`  | No       | Counter token captured from the format string when present. Both `MICRO` and `PATCH` map to the same numeric counter. |

## VersionSourceProvider

| Property       | Type                                     | Required | Description                                                                      |
| -------------- | ---------------------------------------- | -------- | -------------------------------------------------------------------------------- |

| `manifestFile` | `string`                                 | Yes      | Default manifest filename this provider handles.                                 |
| `exists`       | `(cwd: string) => boolean`               | Yes      | Returns `true` when the manifest file exists in `cwd`.                           |
| `getVersion`   | `(cwd: string) => string`                | Yes      | Reads the version string from the manifest. Throws if missing or unreadable.     |
| `setVersion`   | `(version: string, cwd: string) => void` | Yes      | Writes a version string back to the manifest. Throws if the file does not exist. |
